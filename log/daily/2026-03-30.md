### Incident Auto-Closure Analysis

**Problem:** Non-system-wide incidents get stuck in Solved status and never reach Closed.

**Root cause discovered:**
- Root Cause Categorization is mandatory at Solved transition (needed for CFR calculation)
- Solved = customer unblocked (recovery phase complete)
- Teams see Solved as "done" — no motivation to move to Closed
- Closed is perceived as administrative overhead

**Key insight from documentation:**
- Root Cause Categorization marked with [Change Failure Rate ⚡]
- Field determines if incident counts as "change failure" for CFR metric
- Already filled at Solved (mandatory) — investigation done before Solved

**Solution proposed:**
- Auto-close incidents after 7 days in Solved if:
  - System-wide impact = false
  - Root Cause Categorization filled
  - Status = Solved
- Implementation via Jira Automation rule or acli script

**Deliverable:**
- `initiatives/active/incident-auto-closure-analysis.md` — full analysis with implementation options
